<objective>
## Goal
Fix `_setup_virtual_environment` in `create_python_project.py`: Exceptions werden nicht mehr verschluckt.
Editable install (`pip install -e .`) ist mandatory und propagiert Fehler. Dev-Extras-Install ist tolerant.

## Purpose
Beim Anlegen eines neuen Projekts mit `create_python_project` wird das Paket nicht als editable install
im neuen `.venv` installiert, wenn `pip install -e .` fehlschlägt — ohne dass der User einen Fehler sieht.
Root cause: Exception-Swallowing in `_setup_virtual_environment`. Der Fix stellt sicher, dass Fehler beim
editable install sichtbar sind und den Prozess abbrechen.

## Output
- `bin/create_python_project.py` — `_setup_virtual_environment` umstrukturiert: zwei Schritte, klar getrennte Fehlerbehandlung
- `tests/test_create_python_project.py` — TDD-Tests für die Fehlerbehandlung
</objective>

    Ziel (korrekt):
    ```python
    def _setup_virtual_environment(self, target_directory: Path) -> None:
        """..."""
        venv_path = target_directory / ".venv"
        # Venv erstellen und pip upgraden — mandatory, Fehler propagiert
        subprocess.run([sys.executable, "-m", "venv", str(venv_path)], check=True, timeout=120)
        basefunctions.VenvUtils.upgrade_pip(venv_path, capture_output=True)
        pip_executable = basefunctions.VenvUtils.get_pip_executable(venv_path)
        # Editable install — mandatory, Fehler propagiert zum Caller
        self._install_editable_package(pip_executable, target_directory)
        # Dev extras — tolerant, Fehler wird geloggt aber nicht re-raised
        self._install_dev_extras(pip_executable, target_directory)

    def _install_editable_package(self, pip_executable: Path, target_directory: Path) -> None:
        """Editable install — mandatory. Raises CreatePythonPackageError on failure."""
        try:
            subprocess.run([str(pip_executable), "install", "-e", "."],
                cwd=target_directory, check=True, capture_output=False)
        except subprocess.CalledProcessError as e:
            raise CreatePythonPackageError(f"Editable install failed: {e}") from e

    def _install_dev_extras(self, pip_executable: Path, target_directory: Path) -> None:
        """Dev/test extras install — tolerant. Logs warning on failure, does not raise."""
        try:
            subprocess.run([str(pip_executable), "install", "-e", ".[dev,test]"],
                cwd=target_directory, check=True, capture_output=False)
        except subprocess.CalledProcessError as e:
            self.logger.warning(f"Dev extras install failed (non-fatal): {e}")
    ```
