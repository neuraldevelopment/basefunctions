<objective>
## Goal
Fix `update_packages.py` so that packages installed as editable (`pip install -e .`) are never overwritten, regardless of the current working directory (CWD).

## Purpose
The current CWD-based detection of "own package" is fragile: if the developer has a dev-venv active but is in a different directory, the editable install gets replaced by the deployed version. This breaks the development workflow.

## Output
- `bin/update_packages.py` — fixed detection logic in `update_single_venv()` and `update_all_deployments()`
- `tests/bin/test_update_packages.py` — new tests covering the corrected venv-path-based detection
</objective>

    ```python
    editable_packages = set()
    for pkg_name in available_local:
        pkg_dev_paths = _find_development_path(pkg_name)
        for dev_path in pkg_dev_paths:
            dev_path_resolved = Path(dev_path).resolve()
            # Venv liegt innerhalb des dev-dirs → editable install
            if dev_path_resolved in venv_path.parents:
                editable_packages.add(pkg_name)
                break
    ```

    ```python
    to_check = [pkg for pkg in installed if pkg in available_local and pkg not in editable_packages]
    ```

    VORHER: `to_check = [pkg for pkg in installed if pkg in available_local and pkg != package_name]`
    NACHHER: zusätzlich venv-path gegen dev-paths prüfen für jedes pkg in installed.
    ```python
    def _is_editable_in_venv(pkg_name: str, venv_path: Path) -> bool:
        # Prüft ob pkg_name als editable in venv_path installiert ist
        # via _find_development_path: dev_path in venv_path.parents
        for dev_path in _find_development_path(pkg_name):
            if Path(dev_path).resolve() in venv_path.parents:
                return True
        return False

    to_check = [
        pkg for pkg in installed
        if pkg in available_local
        and pkg != package_name
        and not _is_editable_in_venv(pkg, venv_path)
    ]
    ```
