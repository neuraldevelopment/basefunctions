    **REMOVE (debug noise):**
    - Delete every `logger.debug(...)` call unconditionally — debug logs must not exist in production code

    **ADD (error visibility) — apply to each file individually:**
    - In every `except` block that catches an exception AND does not immediately re-raise it:
      Add `logger.error("...", exc_info=True)` BEFORE any fallback logic
      Use `logger.warning(...)` instead if the exception is expected/recoverable (e.g., file not found, optional feature)
    - In every `except` block that re-raises after doing something (logging already must exist):
      Ensure a log call is present before the re-raise
    - Before every `raise ValueError(...)` for invalid user input:
      Add `logger.warning("Invalid input: ...")` with the same message as the exception
    - Before every `raise RuntimeError(...)` or equivalent for internal errors:
      Add `logger.error("Internal error: ...")` with context

    **LOG LEVEL GUIDE:**
    - `logger.error(...)` — unrecoverable failures, unexpected exceptions, data corruption
    - `logger.warning(...)` — recoverable issues, invalid input, optional features unavailable
    - `logger.info(...)` — DO NOT add new info logs (not in scope)
    - `logger.debug(...)` — REMOVE all, do not add new ones

    **FORMAT:** Log messages must include context:
    - Good: `logger.error("Failed to parse argument '%s': %s", arg_name, e)`
    - Bad: `logger.error("Error")`

    Each file must have `logger = logging.getLogger(__name__)` in the LOGGING section.
    Increment patch version in file header (e.g., v1.0.0 → v1.0.1).
    Add log entry to file header: `vX.X.X : Logging audit - removed debug logs, added error/warning coverage`
