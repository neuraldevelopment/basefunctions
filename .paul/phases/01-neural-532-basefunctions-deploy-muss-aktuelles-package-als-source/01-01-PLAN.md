<objective>
## Goal
Fix `update_single_venv()` so that the current package (the one whose `.venv` is
active) is never installed from the deploy directory.

## Purpose
Wenn ein User in `~/Code/neuraldev/<pkg>` arbeitet und `update_packages` ausführt,
darf `<pkg>` selbst nicht aus `~/.neuraldevelopment/packages/<pkg>` installiert
werden. Aktuell passiert genau das, weil `_is_editable_in_venv()` fehlschlägt
(sucht Package-Verzeichnis eine Ebene zu flach). Code-Änderungen am aktuellen
Package werden dadurch vom eigenen venv nicht wahrgenommen.

## Output
- `bin/update_packages.py`: `update_single_venv()` schließt das aktuelle Package aus
- `tests/bin/test_update_packages.py`: Neuer Testfall deckt den Fix ab
</objective>
