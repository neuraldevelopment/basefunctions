<objective>
## Goal
Add a `ConfigCommand` to the CLI subpackage that outputs the current system configuration. The command supports `config` (all packages) and `config <package>` (single package section) as subcommands.

## Purpose
Operators and developers need a quick way to inspect the active configuration at runtime without digging into JSON files. The CLI's `config` command provides this via the existing `ConfigHandler.get_config_for_package()` API.
