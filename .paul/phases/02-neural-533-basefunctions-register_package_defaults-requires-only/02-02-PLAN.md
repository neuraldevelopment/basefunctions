<objective>
## Goal
Update config subpackage documentation (user + system) to reflect the simplified
`register_package_defaults(package_name)` API introduced in v3.6, and create a
demo showing App-perspective usage of the new config system.

## Purpose
After neural-533 (plan 02-01), the `register_package_defaults` signature was simplified
from 2 args to 1 arg (path resolved internally). Both the user and system docs still
document the old 2-arg form with `get_runtime_config_path`. This creates confusion for
App developers and package authors. A working demo is also missing entirely.

## Output
- `docs/basefunctions/config.md` — user docs updated to v3.6 API
- `~/.claude/_docs/python/basefunctions/config.md` — system docs updated to v3.6 API
- `demos/demo_config.py` — runnable end-to-end demo of the config system
</objective>
